# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by opening a security advisory on GitHub or contacting the repository owner directly.

Do not disclose vulnerabilities publicly until they have been addressed by the maintainers.

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | Yes                |

## Best Practices

- Keep your session credentials private
- Do not commit `.env` or `session/` folder contents
- Use environment variables for sensitive configuration
